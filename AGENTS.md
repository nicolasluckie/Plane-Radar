# Agent Instructions — Plane Radar

This file defines rules for AI agents (Cascade, Copilot, etc.) contributing to this repository.
**All commit messages must follow the Conventional Commits specification.**
This is required for automated CHANGELOG generation via git-cliff.

---

## Commit Message Format

```
<type>[optional scope]: <short description>

[optional body — wrap at 72 chars]

[optional footer(s)]
```

- **Subject line**: imperative mood, lowercase after the colon, no trailing period, ≤72 chars
- **Body**: explain *what* and *why*, not *how*; separate from subject with a blank line
- **Breaking changes**: add `BREAKING CHANGE: <description>` in the footer, or append `!` after the type: `feat(api)!: rename endpoint`

---

## Types

| Type | Use for | Appears in CHANGELOG? |
|------|---------|----------------------|
| `feat` | New user-facing features | ✅ Added |
| `fix` | Bug fixes | ✅ Fixed |
| `refactor` | Code restructuring, no behaviour change | ✅ Changed |
| `perf` | Performance improvements | ✅ Changed |
| `docs` | Documentation only | ✅ Changed |
| `test` | Adding or updating tests | ❌ |
| `chore` | Maintenance, deps, tooling | ❌ |
| `ci` | CI/CD pipeline changes | ❌ |
| `style` | Formatting, whitespace | ❌ |
| `build` | Build system changes | ❌ |

## Scopes (optional but encouraged)

| Scope | Covers |
|-------|--------|
| `display` | Framebuffer driver, display init |
| `radar` | Radar rendering, range, theme |
| `adsb` | ADS-B client, aircraft parsing |
| `web` | Flask server, radar template |
| `config` | Environment variable loading/validation |
| `location` | LocationStorage |
| `tests` | Test suite |
| `ci` | GitHub Actions workflows |
| `deps` | Dependency updates |

## Examples

**Good:**
```
feat(web): add dark mode toggle to radar view
fix(adsb): handle missing alt_baro field without crashing
refactor(config): move load_dotenv call inside load_config
chore(deps): bump ruff to 0.15.18
docs: add Dev Setup section to README
test(adsb): add unit tests for fetch_update mock path
```

**Bad:**
```
Updated stuff                   # no type, vague
fix: Fixed the bug              # capitalised after colon, past tense
feat: add new feature.          # trailing period
CHORE: bump deps                # uppercase type
```

## Versioning

This project uses [Semantic Versioning](https://semver.org/):

- `fix` → patch bump (`1.0.0` → `1.0.1`)
- `feat` → minor bump (`1.0.0` → `1.1.0`)
- `BREAKING CHANGE` footer or `!` suffix → major bump (`1.0.0` → `2.0.0`)

Tags must be prefixed with `v` (e.g. `v1.0.0`) for git-cliff to pick them up.

---

## CHANGELOG

`CHANGELOG.md` is auto-generated by git-cliff on every push to `main`.
Do **not** edit it manually — changes will be overwritten by CI.
